Tag typedef

The interface that TagCollection implements.

Used for bit-docs-js/process/codeAndComment and bit-docs-js/process/code.

Note: This is very likely to change to something better.

Object

Options

  • codeMatch {function(codeLine)}:

    Returns true if this tag should process this line of code.

    bit-docs-js/process/code passes the line of code after a comment block to each tag's codeMatch function. The first tag whose codeMatch returns true will have its code method called with the same line.

  • code {function(codeLine, scope, docMap)}:

    Returns properties that should be set on the comment's DocObject.

  • codeScope {Boolean}:

    If code(codeLine) returns a DocObject, set this object as the new scope.

  • add {function(line, curData, scope, docMap, currentWrite, options)}:

    The add function is called when a comment line starts with @ followed by the tag name.

    An add function can do things like:

    • Add directly to the DocObject.
    • Push something onto the stack:
      • ['push', data]
    • Pop some data off the stack:
      • ['pop', String]
    • Change the default property lines are written to
      • ['default', propName]
    • Add to DocMap:
      • ['add', docObject].
    • Change the scope:
      • ['scope', scopeDocObject]
    • Change the scope and the DocObject being added to:
      • ['scope', scopeDocObject, docObject]

    The stack is used to handle tags that might be nested, for example @codestart and @codeend. It provides something to collect data until the closing tag, and have the compiled data "popped" to the parent.

  • addMore {function(line, curData, scope, docMap)}:

    addMore is called if add returns an object.

  • end {function(line, curData, scope, docMap)}:

    end is called on a tag when the comment switches to another tag.
